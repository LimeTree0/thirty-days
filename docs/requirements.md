# 요구사항 명세

> 이 문서는 thirty-days가 **무엇을 만드는지** 정의한다.
> 여기 없는 기능은 만들지 않는다. 추가하고 싶으면 이 문서를 먼저 고친다.
> 상태: **v0.1 확정** — 미결 항목 없음. 변경하려면 이 문서를 고치는 PR부터.

**최종 갱신**: 2026-07-11

---

## 1. 서비스 정의

개발 질문을 올리면 다른 사용자들이 답변하고, 투표와 채택으로
좋은 답이 위로 올라오는 Q&A 서비스. 토론이 아니라 **정답 찾기**가 목적이며,
답변은 시간순이 아니라 평가순으로 정렬된다.

## 2. 용어

| 용어 | 정의 |
|------|------|
| 질문 | 제목 + 본문 + 태그로 구성된 글 |
| 답변 | 특정 질문에 종속된 글. 한 질문에 여러 개 가능 |
| 투표 | 질문 또는 답변에 대한 up/down 평가 |
| 채택 | 질문자가 답변 하나를 정답으로 지정하는 행위 |
| 태그 | 질문의 분류 라벨 (예: `java`, `spring`) |
| 평판 | 투표 결과가 누적된 사용자 점수 |

## 3. v0.1 기능 요구사항

### 3.1 계정

- 회원가입, 로그인, 로그아웃
- 인증 방식: **이메일 + 비밀번호 자체 구현을 먼저** 진행한다.
  OAuth(GitHub 등)는 이후 추가한다.
- 비로그인 사용자도 모든 콘텐츠를 **열람할 수 있다**.
  쓰기(질문, 답변, 투표, 채택)는 로그인 사용자만 가능하다.

### 3.2 질문

- 등록: 제목, 본문, 태그 1개 이상
- 조회: 단건 상세. **조회수를 기록한다.**
- 목록: 페이지네이션 지원. 방식(offset vs cursor)은 목록 구현 착수 시
  ADR로 결정한다.
- 목록 정렬: 최신순 / 투표순 전환 가능. **기본값은 최신순.**
- 수정: 작성자 본인만
- 삭제: 작성자 본인만. 단, **채택된 답변 또는 up표를 받은 답변이 달린
  질문은 삭제할 수 없다.** (SO 정책 준용)

### 3.3 답변

- 작성: 로그인 사용자, 한 질문에 여러 답변 가능
- **자기 질문에 자기 답변 허용** (자문자답)
- 정렬: 채택된 답변 최상단 고정, 나머지는 투표순
- 수정·삭제: 작성자 본인만

### 3.4 투표

- 대상: 질문, 답변
- 종류: up / down
- 규칙: 1인 1글 1표, 자기 글 투표 금지
- **투표 변경(up↔down)과 취소를 허용한다.**
  변경·취소 시 평판도 그에 맞게 재반영된다.

### 3.5 채택

- 질문자만 가능, 질문당 1개
- 채택된 답변은 목록에서 시각적으로 구분되고 최상단 고정
- **채택 취소 및 다른 답변으로의 변경을 허용한다.** (SO 정책 준용)
- 자기 답변 채택: **질문 등록 48시간 후부터 허용한다.** (SO 정책 준용 —
  자문자답 즉시 채택으로 평판 +15를 챙기는 어뷰징 방지)

### 3.6 태그

- 질문 등록 시 1개 이상, **최대 5개** 부착
- **자유 생성 가능**하되, 사전 정의된 태그 목록도 제공한다
- 태그 클릭 시 해당 태그의 질문 목록

### 3.7 검색

- 키워드로 질문 검색
- 검색 범위: **제목만 / 제목+본문** 두 모드를 모두 지원한다
- 태그 필터와 키워드 검색의 조합은 검색 기능 구현 단계에서
  추가 논의한다 (착수 시 이슈로 등록)

### 3.8 평판

- v0.1에서는 **점수 표시만** 한다. 권한 연동은 범위 밖.
- 점수표 (SO 정책 준용):

| 사건 | 대상 | 변동 |
|------|------|------|
| 질문이 up표를 받음 | 질문 작성자 | +10 |
| 답변이 up표를 받음 | 답변 작성자 | +10 |
| 답변이 채택됨 | 답변 작성자 | +15 |
| 답변을 채택함 | 질문 작성자 | +2 |
| 질문/답변이 down표를 받음 | 작성자 | −2 |
| 답변에 down표를 행사함 | 투표자 | −1 |

### 3.9 본문 형식

- 질문·답변 본문은 **마크다운을 지원한다** (코드 블록 포함)

## 4. 비기능 요구사항

- **SLO**: 간주곡 시점에 `docs/slo.md`로 별도 작성 (연기 결정: 2026-07-11)
- **시뮬레이션 적합성**: v0.1의 모든 핵심 행동(가입, 질문, 답변, 투표, 채택,
  검색)은 API로 수행 가능해야 한다. 2막의 OASIS 에이전트가 API 클라이언트로
  동작하기 위한 전제 조건.

## 5. v0.1 범위 밖 (추후 구현)

댓글, 편집 이력, 평판-권한 연동, 중복 질문 닫기, 알림, 프로필 페이지,
OAuth 로그인.

이 목록의 기능은 v0.1 기간 중 구현하지 않는다. 논의가 필요해지면
이슈로 등록하고 다음 마일스톤에서 판단한다.
