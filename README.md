# thirty-days
> 이 게시판은 30일간 SLO를 지키며 생존하는 것이 목표다.

Stack Overflow 스타일의 Q&A 게시판을 만들고, 도메인을 사서 인터넷에 공개한 뒤,
스스로 선언한 SLO(Service Level Objectives)를 지키며 **30일간 운영**하는 프로젝트입니다.

만드는 것은 절반이고, 나머지 절반은 운영입니다. 트래픽은 LLM 페르소나 에이전트와 부하 생성기가 만들며, 매주 의도적인 장애 이벤트(트래픽 스파이크, 스팸 봇 습격, 디스크 고갈)를 일으키고 복구합니다.

## Why

요즘 AI를 활용하여 개발을 하다보니 이게 내 실력인지 AI가 도와줘서 그런건지 확신이 서지 않았습니다. 그래서 AI를 빼고 해도 문제를 해결할 수 있을까?라는 생각에서 이프로젝트를 시작하게 되었습니다. 이 프로젝트의 목표는 현재 내가 가지고 있는 실력이 실제 운영 및 개발 환경에서 어느 정도에 위치해 있는지 부족한 부분이 어디인지를 확인하고 학습해 나가기 위함입니다.

## 구조

| 단계 | 기간 | 내용 |
|------|------|------|
| **1막 — 구축** | 4~6주 | Q&A 게시판 백엔드 구현. 프론트엔드는 AI(Claude Code)가 OpenAPI 계약 기반으로 병렬 개발 |
| **간막 — 공개** | 1~2주 | 도메인 구매, VPS 구축, TLS, 무중단 배포 파이프라인 |
| **2막 — 운영** | 30일 | 공개 상태로 SLO 준수 운영. 시뮬레이터 상시 트래픽 + 주간 이벤트 데이 |

## 규칙

- **1막의 백엔드는 AI 도움 없이 작성한다.** 문서와 레퍼런스만 사용한다.
- **모든 주요 결정은 ADR로 남긴다.** 문서화되지 않은 합의는 존재하지 않는 것으로 간주한다.
- **장애 발생 시, 고치기 전에 가설을 먼저 기록한다.**
- 프로젝트의 상태는 [STATUS.md](STATUS.md)가, 결정의 역사는 [docs/adr/](docs/adr/)이 원본이다.
